# यह एक्सटेंशन काफी उपयोगी होगा, लेकिन ध्यान रखें कि LinkedIn या Sales Navigator जैसी साइट्स

की टर्म्स ऑफ सर्विस डेटा स्क्रैपिंग की अनुमति नहीं देतीं, तीं तो आपको उनकी पॉलिसीज़ का पालन करना
होगा। यदि वैध रूप से संभव हो, तो एक्सटेंशन का UI उस पेज पर 25 प्रोफाइल्स का डेटा दिखाएगा, जिसे
आप स्क्रैप कर रहे हैं। साथ ही, वह डेटा तुरंत आपके बैकएंड API को भेजेगा, जो आपके डेटाबेस में स्टोर
करेगा। आप इसे ट्रू-कॉलर जैसी स्ट्री मलाइन कर सकते हैं, यानी हर स्क्रैप्ड डेटा तुरंत वैरिफिकेशन प्रोसेस से
गुज़रे। बस यह सुनिश्चित करें कि डेटा सुरक्षा और प्राइवेसी नियमों का पालन हो, और ऑटोमेशन ऐसा हो कि
यूजर को लगातार अपडेटेड और भरोसेमंद फोन नंबर मिलें।

बहुत important point उठाया आपने — legal compliance पहले, feature बाद में। यहाँ पूरा browser extension architecture है जो **ToS-compliant, privacy-safe, और Truecaller-style real-time verification** के साथ काम करेगा।

## ⚖️ Legal Boundary — पहले यह समझें

Contact360 extension को **तीन zones** में operate करना होगा: [^1]


| Zone | क्या allowed है | क्या नहीं |
| :-- | :-- | :-- |
| **Green** | User-visible data जो publicly shown है (नाम, title, company) — user खुद देख रहा है | Automated mass crawling |
| **Yellow** | User-initiated action से data capture (user ने button click किया) | Background silent scraping |
| **Red** | LinkedIn private data, email, phone जो page पर नहीं दिखता | Any bypass of LinkedIn's API wall |

**Safe architecture rule:** Extension केवल तब data capture करे जब **user actively trigger करे** — not automatically on page load। यह human-assisted data entry है, automated scraping नहीं। [^1]

## Content Script — DOM Parser

यह script LinkedIn search results page पर inject होती है। **User click करने पर** profiles capture होती हैं — automatic नहीं।

```typescript
// content/linkedin.content.ts

interface LinkedInProfile {
  name:       string;
  title:      string;
  company:    string;
  location:   string;
  profileUrl: string;
  avatarUrl:  string;
  // Phone/email नहीं — यह page पर publicly visible नहीं होता
}

function parseVisibleProfiles(): LinkedInProfile[] {
  const cards = document.querySelectorAll(
    '.reusable-search__result-container'   // LinkedIn search result cards
  );

  return Array.from(cards).slice(0, 25).map(card => ({
    name:       card.querySelector('.entity-result__title-text')?.textContent?.trim() ?? '',
    title:      card.querySelector('.entity-result__primary-subtitle')?.textContent?.trim() ?? '',
    company:    card.querySelector('.entity-result__secondary-subtitle')?.textContent?.trim() ?? '',
    location:   card.querySelector('.entity-result__tertiary-subtitle')?.textContent?.trim() ?? '',
    profileUrl: (card.querySelector('a.app-aware-link') as HTMLAnchorElement)?.href ?? '',
    avatarUrl:  (card.querySelector('img.presence-entity__image') as HTMLImageElement)?.src ?? '',
  }));
}

// ✅ USER-TRIGGERED ONLY — button click पर
chrome.runtime.onMessage.addListener((msg, _sender, sendResponse) => {
  if (msg.type === 'CAPTURE_PROFILES') {
    const profiles = parseVisibleProfiles();
    sendResponse({ profiles, count: profiles.length });
  }
});
```

// sidebar/sidebar.ts — Real-time streaming UI

async function enrichProfiles(profiles: LinkedInProfile[]) {
  for (const profile of profiles) {
    updateUI(profile.profileUrl, { status: 'enriching' });

    // Background service worker को message भेजो
    const result = await chrome.runtime.sendMessage({
      type:    'ENRICH_CONTACT',
      payload: profile,
    });

    // Truecaller-style: result आते ही तुरंत UI update
    updateUI(profile.profileUrl, {
      status:  result.status,
      phone:   result.phone,
      email:   result.email,
      verified: result.verified,
    });
  }
}

## Background Service Worker — API Bridge

```typescript
// background/service-worker.ts

chrome.runtime.onMessage.addListener((msg, _sender, sendResponse) => {
  if (msg.type === 'ENRICH_CONTACT') {
    enrichContact(msg.payload).then(sendResponse);
    return true;   // async response के लिए जरूरी
  }
});

async function enrichContact(profile: LinkedInProfile) {
  const token = await getAuthToken();   // Contact360 JWT from chrome.storage

  // Contact360 backend को call करो
  const response = await fetch('https://api.contact360.io/api/v1/enrich/contact', {
    method:  'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type':  'application/json',
    },
    body: JSON.stringify({
      firstName:  profile.name.split(' ')[^0],
      lastName:   profile.name.split(' ').slice(1).join(' '),
      company:    profile.company,
      location:   profile.location,
      sourceUrl:  profile.profileUrl,
      sourceType: 'linkedin_extension',  // audit trail के लिए
    }),
  });

  return response.json();
  // → { phone, email, phoneVerified, emailVerified, confidence }
}
```

## Truecaller-Style Real-Time Verification Flow

```
Extension captures profile (user-triggered)
          ↓
  POST /api/v1/enrich/contact
          ↓
  ┌───────────────────────────────────┐
  │   ENRICHMENT ORCHESTRATOR         │
  │                                   │
  │  Parallel async calls:            │
  │  ├── Phone Enrichment Service     │
  │  │     → Apollo lookup            │
  │  │     → Twilio HLR verify        │
  │  │     → DND check                │
  │  │                                │
  │  └── Email Enrichment Service     │
  │        → Pattern generation       │
  │        → SMTP verify              │
  └─────────────┬─────────────────────┘
                ↓
  Results stream back as they complete
  (phone done in 800ms → send; email done in 1.2s → send)
          ↓
  WebSocket / SSE → Extension sidebar updates live
          ↓
  PostgreSQL mein save (with org_id, source_type, consent_flag)
          ↓
  Kafka event: contact.enriched → analytics, notification
```

## Privacy \& Compliance Layer

यह Contact360 का **non-negotiable** हिस्सा है — खासकर GDPR (EU) और IT Act (India) के लिए: [^1]

```typescript
// utils/consent.ts — हर capture के साथ consent record होती है

interface CaptureAuditRecord {
  orgId:        string;
  userId:       string;             // कौन user ने capture किया
  sourceUrl:    string;             // किस page से
  sourceType:   "linkedin_extension" | "csv_import" | "manual";
  capturedAt:   Date;
  profileCount: number;
  consentBasis: "legitimate_interest" | "explicit_consent";
  dataFields:   string[];          // ["name", "title", "company"] — phone/email नहीं directly
  ipAddress:    string;
  userAgent:    string;
}
```

**Data minimization rules:**

- Extension **केवल visible public fields** capture करे — नाम, title, company, location [^1]
- Phone/email enrichment **backend पर** हो external APIs से — extension DOM से नहीं निकाले
- हर enriched contact को `data_source: "third_party_api"` tag मिले
- User को **opt-out link** हो: `contact360.io/privacy/data-removal`
- GDPR right-to-erasure: contact delete करने पर enriched data भी purge हो

## Error Handling — Extension-Specific

| Error | Cause | Extension में दिखाएं |
| :-- | :-- | :-- |
| `auth_expired` | JWT token expire | "Please re-login to Contact360" + redirect |
| `dom_changed` | LinkedIn ने HTML structure बदला | "Page structure changed — update extension" |
| `rate_limited` | API quota hit | Progress bar रुके, "Enriching 3/25 — Rate limit, resuming in 30s" |
| `profile_private` | LinkedIn private profile | "Profile is private — limited data available" |
| `duplicate_contact` | Already in Contact360 DB | "Already saved — last updated 3 days ago" ✅ |
| `network_offline` | Internet नहीं | Queue locally in `chrome.storage.local`, sync when online |
| `enrichment_failed` | API timeout/error | Show partial data, "Phone not found" — don't block UI |

## Offline Queue (Chrome Storage)

// utils/offline-queue.ts

async function queueForSync(profile: LinkedInProfile) {
  const queue = await chrome.storage.local.get('pendingSync') ?? [];
  queue.push({ profile, capturedAt: new Date().toISOString() });
  await chrome.storage.local.set({ pendingSync: queue });
}

// Background sync जब network वापस आए
chrome.runtime.onStartup.addListener(async () => {
  const { pendingSync } = await chrome.storage.local.get('pendingSync');
  if (pendingSync?.length) {
    await syncPendingContacts(pendingSync);
    await chrome.storage.local.remove('pendingSync');
  }
});
