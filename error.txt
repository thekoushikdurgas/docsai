# Django CI log — analysis (contact360.io/admin)

## Error (fatal)
- **Message:** `working directory '.../contact360.io/admin'. No such file or directory`
- **Cause:** Workflow used **`defaults.run.working-directory: contact360.io/admin`**. Repo **`thekoushikdurgas/docsai`** is a **standalone** admin project: **`manage.py`** lives at the **repository root**, not under **`contact360.io/admin/`** (that path only exists in the **contact360** monorepo).

## Fix (in repo)
- **`django-ci.yml`:** First step detects layout (`manage.py` in `.` vs `contact360.io/admin/`), sets **`ADMIN_DIR`** in **`GITHUB_ENV`**, then each **`run`** uses **`working-directory: ${{ env.ADMIN_DIR }}`**.

- None material; job name **test** is the workflow job id.
