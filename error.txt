# Deploy Admin to EC2 — log analysis (docsai)

## Error (fatal)
- **Message:** `fatal: not a git repository (or any of the parent directories): .git` (exit 128)
- **Where:** Remote step after `cd ${{ env.PROJECT_DIR }}` (default `/home/ubuntu/docsai`).
- **Meaning:** That directory on EC2 is **not** a **`git clone`** (empty folder, rsync copy, or wrong path). `git fetch` requires a `.git` directory.

## Resolve (server — do once)
SSH to the instance and either:

**A)** Clone into the path the workflow uses (default `/home/ubuntu/docsai`):

```bash
sudo mkdir -p /home/ubuntu/docsai
sudo chown ubuntu:ubuntu /home/ubuntu/docsai
cd /home/ubuntu/docsai
git clone https://github.com/thekoushikdurgas/docsai.git .
```

**B)** Or if the app already lives elsewhere, set GitHub **repository variable** **`EC2_PROJECT_DIR`** to that **absolute** path (Settings → Actions → Variables). The workflow uses `${{ vars.EC2_PROJECT_DIR || '/home/ubuntu/docsai' }}`.

## Warnings / noise in log
- **`##[debug]`** lines are normal Actions diagnostics.
- **Verify deployment** skipped because the deploy step failed (`success()` was false).

## Code changes (repo)
- **`deploy.yml`:** Optional **`vars.EC2_PROJECT_DIR`**, `.git` check with a clearer **`::error::`** message before `git fetch`.
