## Cross-service risk matrix

| Risk | Services | Mitigation owner |
| --- | --- | --- |
| Doc drift on SN endpoints/contracts | salesnavigator, Appointment360, docs/backend/apis | Platform + SN |
| Duplicate writes on retries | extension, salesnavigator, connectra, jobs | Extension + Data |
| PII leakage in telemetry/prompts | logs.api, contact.ai, extension UI | Security + AI |
| Burst load after large SN imports | salesnavigator, connectra, emailapis, mailvetter | SRE + service owners |
| UX mismatch vs backend semantics | extension, app, Appointment360 | Frontend + API |

## Micro-gate template (per patch)

1. Contract diff merged or N/A documented
2. Automated tests or smoke for touched services
3. Telemetry or log evidence for new paths
4. Runbook or on-call note if behavior changed
5. Owner sign-off in era folder or ticket
