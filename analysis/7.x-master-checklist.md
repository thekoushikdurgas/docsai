# 7.x master checklist — patch ladder (`7.n.X` discipline)

**Purpose:** Shared release discipline for every **minor** in `docs/7. Contact360 deployment/` under the `7.x` deployment/governance era. Each minor owns its theme in `7.0 — Deployment era baseline lock.md` … `7.9 — 80 RC Fortress.md` and must implement the corresponding `7.n.x` patch ladder rows.

This file is modeled after `docs/3. Contact360 contact and company data system/3.x-master-checklist.md`, but specialized for **deployment/governance** work:

- RBAC and role-aware access control
- Service-level authorization and audit evidence
- Tenant isolation and policy scoping
- Security posture and secrets hardening
- Observability, release evidence, and RC gates

---

## How to use

1. Pick the target release `7.Y.Z`.
2. Find minor `Y` in `version_7.Y.md` (for `7.8`, use the created file `7.8 — Observability Command Stage.md`).
3. Map patch digit `Z` into the patch ladder table below:
   - `7.Y.0` -> row `0`
   - `7.Y.1` -> row `1`
   - ...
   - `7.Y.9` -> row `9`
4. Before ship, satisfy the patch ladder row requirements for **Contract / Service / Surface / Data / Ops** (where that row is applicable), plus any minor-specific checklist items defined in `version_7.Y.md`.

## Cross-cutting release gate (any `7.x.x` patch)

Before promoting a build:

- Role bypass must be impossible: every privileged action must be gated both at gateway and at the service/handler boundary.
- Audit evidence must be emitted for governance-relevant write paths (and must not contain raw PII beyond policy allowances).
- Tenant boundary must be enforced on read and write paths; failures must be safe (no other tenant identifiers).
- Observability must be adequate for operational support: a trace id/correlation id must link the user/admin action to logs/audit outputs.
- Any security/secret changes must include a rollback path and a minimal runbook step list.
- Admin deployment/governance promotion requires `contact360.io/admin` settings forms to be fully wired (placeholder-only settings is not release-complete for governance controls).
