## Immediate execution queue (codebase-derived, high leverage)

These are “stop-the-bleeding” items that repeatedly show up in the codebase analyses and should be treated as **default gates** across `2.x` work:

1. **Appointment360**: remove inline debug file writes in email/jobs modules before any production release.
2. **emailapis/emailapigo**: publish a single provider + status vocabulary registry; add Python/Go parity fixtures.
3. **Mailvetter**: deprecate legacy endpoints; replace in-memory limiter with Redis-backed limiter; move migrations out of app startup; separate webhook secret from API secret fallback.
4. **s3storage**: replace in-memory multipart session store with durable shared state; remove hardcoded worker function name; make `metadata.json` updates concurrency-safe.
5. **logs.api**: align docs and queries to the canonical S3 CSV implementation; define retention + query windows for bulk telemetry.

## P0 release blocker (credential security gate)

Status legend: ✅ Completed · 🟡 In Progress · 📌 Planned

- 📌 Planned: `contact360.io/email` must remove plaintext mailbox credential persistence and direct credential headers before `2.x` completion.
- 🟡 In Progress: Required migration: token-exchange mailbox session model (short-lived session token + server-side credential storage).

## Data lineage discipline

- **Bulk CSV / outputs:** S3 prefixes + `metadata.json` are the canonical artifact index for bulk flows — see `s3storage-email-system-task-pack.md` and [`email_system.md`](email_system.md).
- **Operational logs:** `logs.api` **S3 CSV** canonical store — see `logsapi-email-system-task-pack.md`.

Minimum expectations to document per minor when touched:

- **S3 key shape** includes at least `job_id` + `user_uuid` (and optional `tenant`) for support/audit.
- **Retention policy** is explicit for CSV objects that may contain PII (encryption + TTL/lifecycle).
- **Metadata freshness**: jobs must either gate on metadata readiness or explicitly handle “metadata not ready yet” states.
