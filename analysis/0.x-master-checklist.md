## Patch micro-gate template (apply at `0.N.P`)

For every patch release, attach a **five-line evidence gate** (Contract / Service / Surface / Data / Ops). Expand in `0.N — <Title>.md` → **Patch ladder**.

| Track | Gate question (must answer Yes or document waiver) |
| --- | --- |
| **Contract** | Did any public or internal API surface change? If yes: diff vs `docs/backend/apis/` or pack; if no: attach “no contract change” note. |
| **Service** | Do critical paths for this patch still boot, health-check, and pass the defined smoke for affected services? |
| **Surface** | Did UI, extension, or admin behavior change? If yes: UX evidence + role checks; if no: note N/A. |
| **Frontend** | Which foundation-era components/routes must render or be scaffolded? List by name or mark N/A. |
| **Data** | Migrations, index mappings, S3 prefixes, or lineage docs updated and linked? |
| **Ops** | Rollback path, secrets, CI step, or runbook delta recorded? |

**Patch intent bands (typical):** `.0` charter · `.1`–`.2` scaffold · `.3`–`.5` hardening · `.6`–`.8` integration/drift · `.9` minor freeze / handoff to `0.(N+1).0`.

## Cross-service risk matrix (foundation — from codebase analyses)

Use this when scoping a minor or patch. Sources: `docs/codebases/*-codebase-analysis.md`.

| Service | Primary risk (0.x) | Mitigation owner |
| --- | --- | --- |
| **Appointment360** (`contact360.io/api`) | Debug file writes; rate limit off by default; idempotency/abuse guard in-memory; schema doc gaps | Platform API |
| **Connectra** | In-memory job queue; broad CORS; ES/PG drift; shared API-key semantics | Platform backend |
| **Jobs** (`contact360.io/jobs`) | Stale `processing` recovery; DAG degree-decrement edge cases; shared `X-API-Key` | Platform backend |
| **s3storage** | Non-durable multipart session state; hardcoded worker function name; `metadata.json` RMW race | Platform + DevOps |
| **emailapis / emailapigo** | Provider + status semantic drift; bulk correctness; observability gaps | Email platform |
| **Mailvetter** | Dual V1/legacy API; in-memory rate limiter; runtime migrations | Email platform |
| **logs.api** | Docs vs implementation (S3 CSV canonical); static API key; query scale | Platform + compliance |
| **contact.ai** | Global API key; enum/schema drift; silent model fallback | AI platform |
| **Sales Navigator** | Global API key; no rate limit; CORS `*`; doc vs route drift | Extension + platform |
| **Email campaign** | Schema drift; unauthenticated routes; nil SMTP auth; dual queue paths | Campaign + DevOps |
| **Admin / DocsAI** | `architecture.md` / `roadmap.md` ↔ Django constants drift (`docs/docsai-sync.md`) | Governance |
