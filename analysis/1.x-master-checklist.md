## Per-patch micro-gate expanded notes

Brief “what the codebase actually ships” notes for the charter patches (`.0`) of each minor. These are used as anchors when expanding the full patch ladder in `version_1.N.md`.

### 1.0.0 — Genesis (User Genesis MVP baseline)

Contract: GraphQL auth (`login`, `register`, `logout`, `refreshToken`, `me`) + credit/usage read (`usage(feature)`) + email finder/verifier contract.

Service: HS256 JWT issuance; logout → `token_blacklist`; credit deduction gate before emailapis calls (`deduct_credit`).

Surface: App entry screens (`/login`, `/register`) + finder/verifier flow + credits badge.

Data: `users`, `credits`, and `token_blacklist` migrations + ledger row lineage.

Ops: Smoke test `signup → first finder → ledger decrement trace` and zero-balance behavior.

### 1.1.0 — Channel (Billing + bulk charter)

Contract: credit-aware job creation payload + billing mutations `subscribe`, `purchaseAddon`, `submitPaymentProof` with idempotency key expectation.

Service: Jobs attach `user_uuid` + billing metadata into `job_node.data`; s3storage stores CSV/proof artifacts; batch emailapis status vocabulary aligned with billing reconciliation.

Surface: Bulk upload UI with job progress + credit warning prior to enqueue.

Data: `job_events` include billing correlation ids; gateway credits/subscription tables updated with batch semantics.

Ops: CSV queue correctness regression suite + success-to-crediting KPI sample (handoff to 1.3).

### 1.2.0 — Bearing (Analytics release charter)

Contract: documented `usage`, `analytics`, and admin GraphQL operations; explicit rate-limit/throttling error shape.

Service: usage aggregation query paths; admin mutations (`creditUser`, `adjustCredits`) wrapped for audit completeness.

Surface: usage dashboard + low-credit/payment feedback cues (e.g., `CreditBudgetAlerts`).

Data: logs.api S3 CSV events for billing/auth so admin views can query operational evidence.

Ops: dashboard health note for rate-limit metrics and MVP scale performance checks.

### 1.3.0 — Keycard (Payment state machine charter)

Contract: idempotency key header for payment-submission and crediting actions.

Service: proof upload persistence (s3storage) + admin review queue + atomic ledger update after `approvePayment`.

Surface: proof upload + success/error toasts; admin approve/decline UX.

Data: `payment_submissions` rows + linkage to `users.uuid`; credits updated only once per correlation.

Ops: duplicate idempotency key failure injection; reconciliation runbook stub for stuck submissions.

### 1.4.0 — Gauge (Usage ledger charter)

Contract: stable `UsageQuery` fields and feature enum alignment with deduction.

Service: usage query avoids N+1 (DataLoaders or bulk joins).

Surface: app usage page pattern + `useUsage`/`usageService` hooks.

Data: ledger rows consistent with deduction (`credits` vs consumed).

Ops: cache invalidation note + ledger consistency spot-checks.

### 1.5.0 — Pulse (Notification rail charter)

Contract: notification semantics (source of truth for low-credit / payment success cues).

Service: balance freshness after billing approve; server-side state correctness.

Surface: `CreditBudgetAlerts` via `MainLayout` + payment success toast path.

Data: idempotent “shown once per session” low-credit policy (no duplicates).

Ops: monitor false-positive rate with correctness KPI.

### 1.6.0 — Switch (Admin control plane charter)

Contract: admin-only resolvers; explicit `require_admin` guards on billing mutations.

Service: `admin_client.py` actions match GraphQL mutations; all adjustments write actor/reason/before-after.

Surface: admin templates with payment submission tables + approve/decline actions.

Data: payment submission state + audit trail storage for credit adjustments.

Ops: reversal runbook for mistaken credit grants and pen-test gate.

### 1.7.0 — Wall (Security hardening charter)

Contract: throttled client error shape (429) and retry guidance for dashboard/extension clients.

Service: enable `GraphQLRateLimitMiddleware` + align with `GraphQLMutationAbuseGuardMiddleware`.

Surface: app backoff UX for 429 responses.

Data: optional persistence of abuse/block counters to logs.api.

Ops: environment-specific tuning and burst/load test to ensure protection without DB exhaustion.

### 1.8.0 — Pack (Credit pack maturity charter)

Contract: document entitlement errors (`PACK_EXPIRED`, `GRACE_PERIOD`) and semantics across APIs.

Service: expiry/lapse/grace transition logic consistent with deduction and billing states.

Surface: expiry banner + renewal CTA and frozen-feature overlay behavior.

Data: `subscriptions` and `credits.reset_at` or equivalent state transitions with migration evidence.

Ops: reconciliation playbook for users stuck in grace and time simulation tests.

### 1.9.0 — Factor (Identity/2FA charter)

Contract: `twoFactor` module schema (setup + verify mutations) and step-up expectations for billing/admin actions.

Service: secure TOTP secret storage + constant-time verify.

Surface: security settings enrollment UX (QR panel, OTP input, recovery code download if enabled).

Data: `two_factor_secrets` or encrypted column on users; audit of enable/disable actions.

Ops: wrong-TOTP negative tests and lockout/support recovery runbook.

### 1.10.0 — Matrix (Ops exit gate service inventory)

Contract: “Billing SLO” one-pager (success rate, crediting latency, ledger error budget) without breaking schema.

Service: synthetic probe `login → usage read` and optional billing env check.

Surface: no new UX, only ops readiness (status/health if exposed).

Data: daily reconciliation report (credits issued vs payments approved).

Ops: billing incident runbook index and backup/restore drill for gateway DB.

**Bands (typical):** `.0` charter · `.1`–`.3` core path · `.4`–`.6` hardening · `.7`–`.8` integration · `.9` minor freeze / handoff.

## Cross-service risk matrix (1.x — billing/credit/user)

| Service | Primary risk (1.x) | Mitigation owner |
| --- | --- | --- |
| **Appointment360** | Wrong credit deduction; billing idempotency gaps; rate limit still off in prod | Platform API |
| **App** | Credit display vs server drift; role-gated billing pages | Frontend |
| **Admin / DocsAI** | Payment approval without audit; constants drift (`docs/docsai-sync.md`) | Governance |
| **Jobs** | Bulk jobs without credit context in metadata | Platform backend |
| **emailapis / emailapigo** | Finder/verifier cost not aligned with gateway ledger; provider status drift | Email platform |
| **Mailvetter** | Plan-tier limits vs billing; API key ownership | Email platform |
| **s3storage** | Payment-proof object policy; PII in uploads | Platform + compliance |
| **logs.api** | Billing events missing or non-queryable; PII in log bodies | Platform + compliance |
| **Connectra** | Heavy search without quota guardrails | Platform backend |
| **contact.ai** | `user_id` FK / cascade; IAM over-privilege | AI platform |
| **Sales Navigator / extension** | Token expiry mid-flow; billing event stub incomplete | Extension |
