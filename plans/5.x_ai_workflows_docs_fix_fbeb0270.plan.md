# 5.x AI Workflows Documentation Fix Plan

## Root Problem

Every `version_5.x.md` file currently contains **6.x Reliability/Scaling era content** (idempotent retries, SLO breach routing, checkpoint resume safety, queue backpressure control). This is doc drift — the content was templated from a different era and never corrected.

Additional issues:

- `emailapis-ai-workflows-task-pack.md` has unresolved template variable `$(System.Collections.Hashtable.era)` in every task
- `ai-workflows.md` uses legacy endpoint prefix `/api/v1/gemini/` (should be `/api/v1/ai/`) and calls stages "4.x" instead of "5.x"
- `ai-cost-governance.md` references "Stages 4.3–4.4" instead of "5.3–5.4", is very thin (12 lines)
- `connectra-ai-task-pack.md`, `jobs-ai-workflows-task-pack.md`, `logsapi-ai-workflows-task-pack.md` are very thin stubs with no actionable checklist items

## Canonical 5.x AI Minor Map

Derived from `[docs/versions.md](docs/versions.md)` and `[docs/roadmap.md](docs/roadmap.md)`:

- **5.0.0 "Neural Spine"** — AI era gate: `contact.ai` Lambda wired, `HFService` + Gemini routing live, basic chat CRUD and `ai_chats` table
- **5.1.0 "Orchestration Live"** — Contact AI integrated in dashboard journeys: chat page, sidebar, model selection, utility mutations live
- **5.2.0 "Explainability Plane"** — `confidence`/`explanation` on `Message` type; `analyzeEmailRisk`, `generateCompanySummary`, `parseContactFilters` live
- **5.3.0 "Spend Guardrails"** — Per-user AI quotas, provider cost caps, rate limiting, credit deduction wired
- **5.4.0 "Prompt Constitution"** — Prompt versioning, `prompt_version` audit field, governance controls

Extended minors (valid AI scope from codebase analysis docs, filling 5.5–5.10):

- **5.5.0 "Signal Enrichment"** — SN contact AI-field quality: `data_quality_score`, `about`, `seniority`, `departments` pipeline for AI context eligibility
- **5.6.0 "Batch Intelligence"** — TKD Jobs AI batch processors: `model`/`confidence`/`cost` envelope, quota-aware scheduling
- **5.7.0 "Artifact Vault"** — S3 AI artifact classes (input/context/output/intermediate), retention/access policy, lineage metadata
- **5.8.0 "Audit Telescope"** — Logs API AI telemetry schema: prompt/tool/response events, PII write guards, cost anomaly alerts
- **5.9.0 "Explainability Export"** — Mailvetter AI-consumable payloads (`top_factors`, `risk_reason`, `confidence_band`) + EmailAPIs AI payload boundaries
- **5.10.0 "Connectra Intelligence"** — Connectra AI field whitelist, VQL AI-safe query contract, enrichment artifact lineage
