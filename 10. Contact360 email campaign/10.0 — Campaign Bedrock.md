# Version 10.0 тАФ Campaign Bedrock

- **Status:** тЬЕ Completed
- **Codename:** Campaign Bedrock
- **Era:** 10.x (Contact360 email campaign)
- **Roadmap:** Foundation hardening for campaign service infrastructure тАФ ships as **`10.0.0`** per [`docs/versions.md`](../versions.md)
- **Summary:** Wire the **campaign backbone** end-to-end: `CampaignWizard` UI тЖТ Appointment360 GraphQL (`campaigns`/`sequences`/`templates` modules) тЖТ `emailcampaign` REST API тЖТ Asynq `campaign:send` worker тЖТ SMTP delivery тЖТ `logsapi` event sink. Establish the `completed_with_errors` terminal state, idempotency key tracking, and suppression list sync with Connectra.
- **Owner:** Email Campaign Engineering + Platform API
- **Patch closure:** Every codenamed patch file includes **Micro-gate** + **Service task slices**. Era hub: [`versions.md`](../versions.md).

## Scope

- **Target:** `10.0.x` patches тАФ stable campaign service and shared contract surfaces before deep slices in `10.1+`.
- **In scope:** Campaign CRUD, sequence GraphQL contracts, template management, Asynq queue state machine, SMTP auth baseline, suppression merge, compliance event fields.
- **Out of scope:** Full audience segmentation maturity (`10.2`); A/B testing (`10.5`); advanced analytics (`10.8`).
- **Owners:** Email Campaign Engineering + Platform API.
