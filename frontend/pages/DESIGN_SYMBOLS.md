# Page design symbols

Shared notation for `docs/frontend/pages/*_page.md` specs: layout, controls, data, and admin-only regions. Use **one composite line** per page in **Page design (symbols)** (see footer blocks in each spec) plus tables already generated for buttons/inputs.

## Layout and regions

| Symbol | Meaning |
| --- | --- |
| `[L]` | App shell: sidebar + header + main content slot (`MainLayout`) |
| `[H]` | Page header / title strip (`PageHeader`, `DashboardHeader`, etc.) |
| `[C]` | Main content column (scrollable) |
| `[Ad]` | Admin-only toolbar, tab, or section |
| `[W]` | Wizard / multi-step flow container |

## Navigation and structure

| Symbol | Meaning |
| --- | --- |
| `[T]` | Tab group ŌĆö name tabs in prose, e.g. `[T: Finder \| Verifier]` |
| `[Q]` | Query / list surface (table, inbox list, log viewer) |
| `[F]` | Form-heavy surface (settings, billing, mapping modals) |
| `[K]` | KPI / metric cards or stat grid |
| `[G]` | Graph or DAG visualization (charts, job execution flow) |

## Interactive controls

| Symbol | Meaning |
| --- | --- |
| `(btn:label)` | Button; optional variant: `(btn.primary:Save)` |
| `(in)` | Single-line text input |
| `(ta)` | Text area |
| `(sel)` | Select / combobox |
| `(cb)` | Checkbox |
| `(rb)` | Radio group |
| `(pb)` | Progress bar (determinate or indeterminate) |
| `(tbl)` | Data table with sort/filter |
| `(md)` | Modal / dialog |
| `(card)` | Card or panel |
| `(toast)` | Toast / inline alert |

## Data and contracts

| Symbol | Meaning |
| --- | --- |
| `{GQL op}` | GraphQL operation name (dashboard app ŌåÆ Appointment360) |
| `{REST}` | REST call (e.g. Mailhub `contact360.io/email`) |
| `{SSE}` | Server-sent events / streaming |

**Backend endpoint links:** Each `*_page.md` can include an `AUTO:endpoint-links` table (generated by `link_endpoint_specs.py`) mapping `graphql/OperationName` to `docs/backend/endpoints/*_graphql.md`. **Naming rules** (including when page specs use shorthand vs canonical operation names) are documented in [../backend/endpoints/ENDPOINT_DATABASE_LINKS.md](../backend/endpoints/ENDPOINT_DATABASE_LINKS.md).

## Era tags (cross-reference)

Map symbols to product phase using **`docs/version-policy.md`** and the registry in **[index.md](index.md#era-map-0x--10x)**. Typical patterns:

- **0.x** ŌĆö `[L]` `[H]` auth shell; minimal `(btn)` `(in)`
- **1.x** ŌĆö `[F]` billing/profile; `(md)` payment; `[K]` usage
- **2.x** ŌĆö `[T]` finder/verifier; `(pb)` job/export progress; `(tbl)` results
- **3.x** ŌĆö `[Q]` contacts/companies; `(tbl)` + filter sidebars
- **4.x** ŌĆö extension surfaces (see `docs/frontend.md`); marketing + LinkedIn app page
- **5.x** ŌĆö `[G]` or stream panels; AI chat `(ta)` + `(btn)`
- **6.x** ŌĆö error/retry `(toast)`; skeletons; analytics `[K]` `[G]`
- **7.x** ŌĆö `[Ad]` governance; role-gated `[T]`
- **8.x** ŌĆö API keys, integrations `[F]` `[Q]`
- **9.x** ŌĆö workspace / connector wizards `[W]`
- **10.x** ŌĆö campaign builder `[W]` `[F]` `(pb)` send progress

## Example composite lines

**Dashboard (`dashboard_page`):**

`[L] > [H] > [K:stats] + [G:performance] + [Q:activity] + [Q:exports] + [K:usage] + (btn:quick-actions) ŌĆö {GQL UserStats,Activities,ListExports,Usage}`

**Billing (`billing_page`):**

`[L] > [H] > [T:User \| Admin] > [F:subscription] + [F:plans] + (md:payment) + (tbl:invoices) ŌĆö {GQL GetBilling,GetPlans,Subscribe,...}`

**Mailhub inbox (`mailhub_inbox_page`):**

`[L] > [Q:folder-list] + [Q:message-rows] > (btn:refresh|compose) ŌĆö {REST}` *(compose route may be planned)*

**Planned dedicated export hub (`export_page`):**

`[L] > [H] > [T:My Exports \| S3 Files] > [Q:export-table] + (pb:row) + (sel:filters) ŌĆö {GQL Exports,GetExport,CancelExport,S3ListFiles}` *(not implemented as `/export` yet; see spec + [index.md](index.md))*

## Regenerating footers on all page specs

After changing **era** or **registry** rows in [index.md](index.md), re-run:

`python augment_page_specs.py` (from `docs/frontend/pages/` or repo root with `python docs/frontend/pages/augment_page_specs.py`).

That refreshes **Era coverage**, **Page design (symbols)**, and **Navigation** inside each `*_page.md` between `<!-- AUTO:design-nav -->` markers.
